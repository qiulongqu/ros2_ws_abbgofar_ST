# ABB GOFAR CRD15000_12_127 环境搭建

该项目仅针对于12_127版本的机器人搭建，与实验室10_152型号有所出入，请注意区分。

# 0. 软件安装

**注意**

在系统环境搭建之前需要安装linux系统,因为ROS2需要linux系统中的22.04版本才可以运行。

如果电脑是linux系统，可以直接去安装ROS2,如果是windows系统，需要安装虚拟机，在虚拟机中安装linux系统，然后在linux系统中安装ROS2。

虚拟机可以使用VMware或者WSL,linux系统推荐使用Ubuntu系统

**注意**

本项目使用的是WSL虚拟机，在其中安装了22.04版本的Ubuntu系统，并安装了ROS2的最新版本Jazzy Jalisco。

# 1. 构建纯净的工作空间与获取官方模型

## 1.1 创建工作空间并拉取官方主仓库：

```bash
source /opt/ros/jazzy/setup.bash
mkdir -p ~/ros2_ws_abbgofar/src
cd ~/ros2_ws_abbgofar/src

# 深度为1拉取官方仓库，节省时间
git clone --depth 1 https://github.com/ros-industrial/abb.git temp_abb
```

## 1.2 提取核心资产并清理垃圾

```bash
# 只提取我们需要的基础资源包和 GoFa 15000 支持包
mv temp_abb/abb_resources ./
mv temp_abb/abb_crb15000_support ./
rm -rf temp_abb
```

# 2. 核心配置文件改造

由于官方的开源支持包目前仍基于 ROS 1 架构,我们需要删掉旧的 ROS 1 配置，并写入符合 ROS 2 规范的 CMakeLists.txt 和 package.xml，同时确保底层依赖链（xacro）被打通。

## 2.1 改造 abb_resources (公共资源包)

```bash
cd ~/ros2_ws_abbgofar/src/abb_resources
rm CMakeLists.txt package.xml

cat << 'EOF' > CMakeLists.txt
cmake_minimum_required(VERSION 3.8)
project(abb_resources)
find_package(ament_cmake REQUIRED)
install(DIRECTORY urdf DESTINATION share/${PROJECT_NAME})
ament_package()
EOF

cat << 'EOF' > package.xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>abb_resources</name>
  <version>1.0.0</version>
  <description>ROS 2 resources for ABB</description>
  <maintainer email="robot@todo.com">Robot Engineer</maintainer>
  <license>Apache-2.0</license>
  <buildtool_depend>ament_cmake</buildtool_depend>
  <export><build_type>ament_cmake</build_type></export>
</package>
EOF
```

## 2.2 改造 abb_crb15000_support (GoFa 15000 支持包)

```bash
cd ~/ros2_ws_abbgofar/src/abb_crb15000_support
rm CMakeLists.txt package.xml

cat << 'EOF' > CMakeLists.txt
cmake_minimum_required(VERSION 3.8)
project(abb_crb15000_support)
find_package(ament_cmake REQUIRED)
install(DIRECTORY meshes urdf DESTINATION share/${PROJECT_NAME})
ament_package()
EOF

cat << 'EOF' > package.xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>abb_crb15000_support</name>
  <version>1.0.0</version>
  <description>ROS 2 support for ABB CRB15000</description>
  <maintainer email="robot@todo.com">Robot Engineer</maintainer>
  <license>Apache-2.0</license>
  <buildtool_depend>ament_cmake</buildtool_depend>
  <exec_depend>abb_resources</exec_depend>
  <exec_depend>xacro</exec_depend>
  <export><build_type>ament_cmake</build_type></export>
</package>
EOF
```

# 3. 基于xacro手动生成URDF模型(可选，setuo assistant不稳定，很可能导致失败)

为了同时避开 MoveIt Setup Assistant 读取复杂宏文件导致的内存崩溃 (SegFault)，以及 ROS 2 启动脚本读取纯静态文件导致的元组参数解析错误 (Tuple Error)，我们需要制作一个“伪装文件”。

提前在/home/yunhao/ros2_ws_abbgofar/src/abb_crb15000_support/package.xml中添加xacro依赖和abb_resources依赖
可能还要添加urdf依赖

## 3.1 编译并刷新环境（让系统认识刚写好的两个包）：

```bash
cd ~/ros2_ws_abbgofar
colcon build
source install/setup.bash
```

## 3.2 手动编译模型并伪装后缀名为xacro的URDF文件：

```bash
cd ~/ros2_ws_abbgofar/src/abb_crb15000_support/urdf

# 1. 将包含复杂逻辑的 .xacro 翻译成纯净的、没有逻辑依赖的 .urdf
xacro crb15000_12_127.xacro > crb15000_12_127_pure.urdf

# 2. 强行将后缀改回 .xacro 以欺骗 ROS 2 启动脚本
cp crb15000_12_127_pure.urdf crb15000_12_127_pure.xacro
```

# 4. 配置 MoveIt 运动规划大枢纽（可选，如果3不成功，4将不会显示任何模型）

## 4.1 启动配置助手

```bash
cd ~/ros2_ws_abbgofar
source install/setup.bash
ros2 run moveit_setup_assistant moveit_setup_assistant
```

## 4.2 在图形界面中严格执行以下配置

### 4.2.1 Self-Collisions（自碰矩阵）

- 点击左侧 Self-Collisions -> 点击主界面的 Generate Collision Matrix。

- (它会自动计算上万次，找出机械臂绝对碰不到一起的零件，为你以后的电脑省下大量算力。)

### 4.2.2 Virtual Joints（虚拟关节）

- 点击 Virtual Joints -> Add Virtual Joint。

- Virtual Joint Name: 填 virtual_joint

- Child Link: 选 base_link

- Parent Frame Name: 填 world

- Joint Type: 选 fixed

- 保存。(这告诉系统机械臂是焊死在地板上的，不会乱飞。)

### 4.2.3 Planning Groups（规划组 - 极其重要！）

- 点击 Planning Groups -> Add Group。

- Group Name: 填 manipulator (一会我们写 Python 代码就靠这个名字唤醒它！)

- Kinematic Solver: 选 kdl_kinematics_plugin/KDLKinematicsPlugin

- 点击底部的 Add Kin. Chain 按钮：

    - Base Link 选 base_link

    - Tip Link 选 tool0 (机械臂的最末端)

- 保存。

### 4.2.4 ROS 2 Controllers（控制器配置 - 为代码控制打基础）

- 点击 ROS 2 Controllers -> 点击 Auto Add Joint Trajectory Controllers For Each Planning Group。

- (这一步会生成接收我们 Python 代码指令的动作服务器。)

### 4.2.5 MoveIt Controllers（动作执行器）

- 点击 MoveIt Controllers -> 点击 Auto Add FollowJointTrajectory Controllers...。

### 4.2.6 Author Information（作者信息）

- 随便填上你的名字（Name）和邮箱（Email）。

- (如果不填，下一步它会拒绝生成文件。)

### 4.2.7 Configuration Files（生成最终包！）

- 点击左侧最后面的 Configuration Files。

- Configuration Package Save Path 点击 Browse，选择路径到你的 src 文件夹：Home -> ros2_ws_abbgofar -> src。

- 在右上角新建一个文件夹，命名为 abb_crb15000_moveit_config，选中它并打开。

- 点击 Generate Package 按钮！

---
想手动编写abb_crb15000_12_127_moveit_config包，最好使用codex等工具，进行氛围编程。
